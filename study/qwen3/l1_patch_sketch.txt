L1 patch sketch — to apply after the debug build finishes.

Goal: in Graph::Activate(), after CreatePrimitivesAndExecConstants
completes, walk all Input nodes and call ov::wsh::Extension::hint_evict
on each Constant. By that point every primitive's prepareWeightsMemory
has run and the packed copies are live; the original mmap pages are
safe to evict.

Why this site and not Input::cloneBlobIfRequired:
- cloneBlobIfRequired runs at graph-Init time, BEFORE prepareWeightsMemory.
  Calling hint_evict there forces page-faults during primitive setup,
  paying compile-time latency to bring pages back.
- Graph::Activate runs once after all primitives are prepared.
  hint_evict here is no-cost during compile and immediate-effect
  for runtime RSS.

Why dynamic_cast and not a virtual method:
- We don't want to add a hint_evict virtual to Node (most nodes have
  no source Constant). One-off behaviour for Input nodes only.

Risks (small):
- Multiple Input nodes can share a Constant if the IR has been
  optimized with constant-folding+sharing. Calling hint_evict
  N times on the same Constant is safe (it's a hint, MADV_DONTNEED
  is idempotent), but redundant.
- The hint is exactly madvise(MADV_DONTNEED) on Linux. On platforms
  where MappedMemory::hint_evict is a no-op (Windows, maybe macOS in
  some configs), the patch is also a no-op. Correctness preserved.

Validation plan once built:
1. Build OV Debug with patched graph.cpp.
2. Set LD_LIBRARY_PATH to point at the built libs.
3. Set PYTHONPATH to point at the built Python bindings.
4. Re-run probe_raw_breakdown.py --T 770 and compare VmHWM to
   the 2211 MiB baseline. Expect ~1500 MiB (or whatever the
   simulated madvise from probe_manual_madvise gave).
5. Re-run at --T 8192. Expect ~7100 MiB (down from 7848).
